# Deferred Notes

Items flagged for a future phase. **Check at the start of each `/plan:phase`.**

## Rules

- Add items during active work (build or test phases) when you notice cross-phase concerns
- Review at start of each `/plan:phase` — incorporate relevant items into the new plan
- Remove entries once addressed (check the box and note which phase resolved it)
- If an item keeps getting deferred across 3+ phases, escalate to user — it may need its own feature scope
- If an item becomes a full feature, move it to a phase shift (`/plan:shift`) rather than leaving it here
